테스트 코드에는 E2E Test -> Integration Test -> Unit Test
이렇게 3개가 있다

Unit test는 단위 테스트이다.
- 함수, 모듀르 클래스 이런식으로 즉 단위 테스트

Integration Test는 통합 테스트
- 하나의 단위가 아니라 여러가지를 통합했을때 이들이 잘 작동하는지 테스트 하는것
즉 여러가지 단위를 모아서 테스트한다는것이다.

E2E Test는 끝과 끝을 테스트한다 즉 UI테스트 (사용자 테스트)
- 사용하자 우리 사이트를 사용했을때 그때의 테스트를 말한다.
즉 사용자가 실행하면서 쓸때 발생할수있는 오류를 테스트한다.

비용이 
unit이 가장 적고, 그 다음 intrgration Test, 그 이후 E2E Test
즉 비용이 올라가고 작성 코드도 복잡해진다.
또한 유닛테스트를 작성하면서 하는것이 개발 효율에 더 좋다.
그래서 스피드 적인 부분도 Unit Test가 간단하기 때문

TDD란
테스트 주도 개발
개발 전 테스트 코드를 먼저 작성하고 개발하는 방식

TDD는 즉 테스트 코드를 먼저 작성하고 테스트만 통과할 정도의 코드만 작성해서 다시 테스트를 수행하고
그 다음 테스트 쭉쭉 그런 형식으로 진행하는 방식이다.

TDD의 장점 
프로젝트의 설계하는 부분에서 더 많은 생각을 하고 테스트 코드를 작성하기 전에 
모든 요규 사항에 대해 점검하고 코드를 작성할수 있다.
