CSR

라우팅이란 새로운 url을 요청하면 그 url에 적합한 브라우저 페이지 전체가 다른 데이터를 받아옴

csr이란 새로운 url을 입력했을때 업데이트가 필요한 부분만 동적인 요청을 한다.

(둘의 차이 전체를 요청할지 혹은 부분적인 내가 원하는 부분만 요청할지 차이다)

장점: history이용 

라이브러리를 공부할때는 항상 공식사이트를 봐라
특히 react router dom 같은 경우는 자주 내용이 변경할수 있기떄문에 공식사이트를 보자

react router dom란
Client side routing
사용자가 링크를 클릭했을때 서버한테 요청하지 않아도 새로운 UI를 보여줄수있다.
사용자에게 빠른 페이지를 보여줄수있다. 예를 들어 페이지에 모든 부분을 받아오는것과 필요한 부분을 받아오는것의 차이라고 생각하자

대신 한 라이브러리를 너무 의존하지 말자. 만약 라이브러리가 변경, 혹은 없어지면 그때 다시 유지보수해야한다.
